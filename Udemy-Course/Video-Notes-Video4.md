k8S = Container Orchestration

Container = most popular = Docker, dazu habe ich ebenfalls ein Udemy-Kurs belegt.

Zusammenfassung Docker:
Ziel ist es eine unabhängige "underlying Infrastructure" zu erstellen. => Mit Docker kann OS ersetzt werden und die Dienste nach Wunsch Version OS-Unabhängig läuft. Ziel ist es die Compatibility über alle Umwelten (OS & Versionen) hinweg. Jede komponente (bspw node / mongo db / mesagging usw)

Container sind komplett isolierte Envs. ähnlich wie Virtualmachine, unterschied ist dass die den gleichen basis kernel teilen. Daher kann auch kein windows in einem Linux emuliert werden. 




Tool um application in Container zu fassen.

Es könnnen von jeder Instnaz beliebig viele Instances gelden werden.

image = Template um einenn oder mehr Container zu erstellen

Container = running image mit eigener Umgebung.

Container Vorteile: in einem Deployment von Developer zu Operations. Man kann einfach ein Docker file erstellen und bereitstellen.
